## Contributing

Thank you for your interest in contributing to Defend API.

### Development setup

- Fork the repository and create a feature branch from `main`.
- Install dependencies with `pip install -r requirements.txt`.
- Run the API locally with `uvicorn defend_api.main:app --reload`.

### Coding standards

- Write small, focused pull requests.
- Add or update tests for any behavior change.
- Keep public APIs backwards compatible where possible; document breaking changes clearly.
- Follow the existing code style and type hints.

### Submitting changes

- Ensure `pytest` passes locally.
- Ensure CI is green before requesting review.
- Update `README.md`, `ARCHITECTURE.md`, or `docs/` when you introduce new features or configuration.
