# Security Policy

Defend helps detect common LLM risks, but it cannot guarantee safety against every possible attack or model failure. If you believe you found a security issue, please report it responsibly.

## Reporting a vulnerability

Preferred: open a **private** report using GitHub’s security features (the **Security** tab / Security advisory flow), if available for this repository.

Alternatively: open an issue using the `Security bug report` template, and keep exploit details non-destructive and minimized until a maintainer responds.

## What to include

- Affected component (API route, module, version, etc.)
- What security impact you believe is possible
- Steps to reproduce (only as much as needed for triage)
- Any relevant logs/output with secrets redacted

## Responsible disclosure

Please do not publish full exploit details publicly until:

1. You have a maintainer response, and
2. A fix or mitigation is available, or the maintainers request more time.

## Timelines

We aim to acknowledge reports quickly and discuss next steps. Exact timelines vary based on severity and complexity.
